📋 STEP-BY-STEP DEPLOYMENT GUIDE:

STEP 1: CREATE GITHUB REPOSITORY (2 minutes)
---------------------------------------------
1. Go to: https://github.com/ENDEVSOLS
2. Click "New repository"

   - Description: RAG retrieval regression testing — define Golden Questions, detect lost chunks in CI
   - Visibility: Public

4. Click "Create repository"

STEP 2: INITIALIZE GIT & PUSH CODE (3 minutes)
-----------------------------------------------
cd /Users/apple/PycharmProjects/longprobe

# Initialize git
git init

# Add all files
git add .

# Create initial commit
git commit -m "feat: initial release of LongProbe v0.1.0

- RAG retrieval regression testing framework
- Golden Questions in YAML
- Three match modes (ID, text, semantic)
- Baseline storage and regression detection
- CLI with multiple output formats
- Pytest plugin integration
- Adapters for Chroma, Pinecone, Qdrant, LangChain, LlamaIndex, HTTP
- Question generation with LLM support
- Auto-capture functionality
- Watch mode for continuous testing
- GitHub Actions reusable workflow
- 172 unit tests (100% passing)

Part of the EnDevSols Long Suite for production RAG systems."

# Add remote
git remote add origin https://github.com/ENDEVSOLS/LongProbe.git

# Push to main
git branch -M main
git push -u origin main

STEP 3: CREATE RELEASE TAG (1 minute)
--------------------------------------
# Create and push tag
git tag -a v0.1.0 -m "Release v0.1.0 - Initial public release"

STEP 6: PUBLISH TO PYPI (5 minutes)
------------------------------------
# Install twine (if not already installed)
pip install twine

# Option A: Test on TestPyPI first (RECOMMENDED)
twine upload --repository testpypi dist/*
# Username: __token__
# Password: <your TestPyPI token from https://test.pypi.org/manage/account/token/>

# Test installation from TestPyPI
pip install --index-url https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple/ longprobe
longprobe --version

# Option B: Upload directly to PyPI
twine upload dist/*
# Username: __token__
# Password: <your PyPI token from https://pypi.org/manage/account/token/>
